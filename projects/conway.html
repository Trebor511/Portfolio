Conway’s Game of Life

.NET Application • C# • Cellular Automata • UI/UX Systems

Overview

My implementation of Conway’s Game of Life was developed as part of Full Sail University’s Data Structures & Algorithms curriculum. The project required building a fully interactive .NET Windows Forms application capable of simulating Conway’s classic cellular automaton while providing a robust suite of user controls, configuration options, and persistent settings.

The final submission includes all required basic features, all advanced features, and a polished UI/UX that emphasizes clarity, responsiveness, and ease of use.

Basic Features

  • Interactive Grid Rendering: Users can toggle cells alive or dead by clicking.
  • Simulation Controls: Start, Pause, and Step (Next) generation controls via menu items and toolbar buttons.
  • Randomization: Universe can be randomly populated using a user-defined seed or time-based logic.
  • Clearing the Grid: A New/Clear option removes all living cells.
  • Saving & Loading: Universe can be stored in a text file and reloaded, including universe size.
  • Status Bar: Displays current generation count and number of living cells.
  • Adjustable Timer: User can configure milliseconds between generations.
  • Configurable Universe Size: Universe dimensions editable through a modal dialog.
  • Neighbor Count Display: Toggleable numeric display for each cell’s neighbor count.
  • Full View Menu: Toggles gridlines, neighbor counts, universe type, and HUD visibility.

Advanced Features

  • Customizable Colors: Users can choose grid, living cell, and background colors via modal dialogs.
  • Boundary Behavior: Supports both finite and toroidal (wrapping) universe types.
  • HUD Overlay: Displays universe size, generation, living cells, and boundary type.
  • Persistent Settings: All adjustable settings (size, speed, colors) persist across sessions.
  • Reset & Reload: Users can revert to default settings or reload previously saved preferences.

Implementation Notes

This project involved extensive use of object-oriented principles, event-driven programming, file I/O, timer management, and UI state synchronization. The persistent settings system required building a custom serialization workflow to ensure user configuration remained intact between launches.

The completed application demonstrates strong fundamentals in .NET development, UI architecture, state management, and algorithmic logic through a fully realized simulation tool.
