Concentrate!

Vulkan API • ECS Architecture • C++ • Custom Gameplay Systems

Overview

Concentrate! is a retro arcade-style game inspired by the classic Tapper! and built entirely without a traditional game engine. Developed using the Vulkan graphics API and the FLECS Entity-Component-System (ECS) framework, this project focuses on low-level rendering, manual memory management, and custom gameplay architecture.

The game reimagines the original Tapper gameplay into a more family-friendly experience—replacing beer with orange juice—while maintaining the rapid, timing-based action of the arcade classic. Sprite rendering, input handling, physics behaviors, entity updates, and core game loop logic were all implemented from the ground up, providing a hands-on understanding of real-time rendering and ECS-driven gameplay.

Technical Features

  • Vulkan Rendering Pipeline: Implemented low-level rendering, swapchain setup, vertex buffers, texture sampling, and synchronization primitives.
  • FLECS ECS Architecture: Designed entities and components for players, drinks, patrons, and game systems, enabling efficient updates and behavior separation.
  • Custom Sprite System: Built a 2D sprite renderer on top of Vulkan's command-buffer architecture for fast and efficient rendering.
  • Game Logic & Input Handling: Implemented movement, drink-serving behavior, patron AI, scoring, and failure states from scratch.
  • Level Structure & Timing Systems: Built a complete gameplay loop including difficulty escalation, spawn timing, and resource management.

Development Notes

Concentrate! built directly on top of my previous Level Renderer project, extending the rendering foundation into a fully playable game. Working with Vulkan and FLECS provided a deep understanding of graphics pipelines, memory handling, and data-oriented architecture.

The project reinforced advanced skills in:

  • Low-level rendering APIs (Vulkan)
  • ECS-based engine architecture
  • Real-time update loops & performance optimization
  • Input systems & game mechanic scripting
  • Memory management and GPU/CPU synchronization

Together, these systems represent a complete custom-built game framework—with no Unity, Unreal, or traditional engine tools involved.
